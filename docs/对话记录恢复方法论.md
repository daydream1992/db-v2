# 对话记录恢复方法论

## 场景
Claude Code 窗口闪退/崩溃，未保存的进度通过对话记录找回。

## 如何找回和接续中断的对话任务

### 步骤

1. **查 last-prompt** — 从当前 session jsonl 中 grep `lastPrompt` 获取用户最后输入的任务描述
2. **查 git status** — unstaged 修改记录了中断前的工作内容
3. **查 git diff** — 对比 HEAD 看具体改动

### 命令

```bash
# 1. 找到当前 session 文件
# session id 在启动时的系统消息里，或者从 sessions 目录查找最新的 json 文件
cd C:/Users/day/.claude/projects/K--DB----v2/
grep lastPrompt 787b99eb-e267-43a7-b7b0-a125560b12c2.jsonl

# 2. 查看未提交的改动（中断前的工作内容）
cd K:/DB数据库_v2
git status --short
git diff
```

### 关键路径

- session 目录：`C:\Users\day\.claude\sessions\`
- 对话记录：`C:\Users\day\.claude\projects\K--DB----v2\{sessionId}.jsonl`
- git diff 显示 unstaged = 中断前未提交的工作
