## 一、快速开始（推荐：离线免端口方式）

### 方式 1：双击打开（最简单）

**前提**：已完成一次静态构建（见下方"首次构建"章节）

1. 进入 `db_scripts_analysis/0_weihuxiezuo/` 目录
2. 双击 `start-dataops.bat`（Windows）或运行 `bash start-dataops.sh`（macOS/Linux）
3. 浏览器自动打开 DataOps 管理台
4. **无需任何端口、无需启动服务器**

或者直接双击 `out/index.html` 文件，即可在浏览器中打开。

### 方式 2：开发模式（需端口 3000）

如需实时修改代码并预览：

```bash
cd db_scripts_analysis/../    # 进入项目根目录
bun run dev                    # 启动开发服务器（端口 3000）
# 浏览器打开 http://localhost:3000
```

## 二、首次构建（离线版）

### 步骤 1：安装依赖

```bash
cd /path/to/my-project        # 进入项目根目录
bun install                    # 安装所有依赖
```

### 步骤 2：构建静态文件

```bash
bun run build:static           # 生成 out/ 目录（约 2MB）
```

构建完成后，`out/` 目录包含完整的 HTML/CSS/JS/字体文件，可完全离线使用。

### 步骤 3：打开

```bash
# 方式 A：直接双击
open out/index.html            # macOS
xdg-open out/index.html        # Linux
start out/index.html           # Windows

# 方式 B：本地 HTTP 服务（更稳定）
bunx serve out -p 8080         # 浏览器打开 http://localhost:8080
```

## 五、核心功能详解

### 5.1 Dashboard 钻取

- **数据表总数** → 点击跳转健康度视图
- **执行成功率** → 点击弹出 7 日成功率明细 Popover
- **入库行数** → 点击弹出 Top 5 大表 Popover
- **待处理告警** → 点击跳转规范校验视图
- **时间范围** → 切换 7d / 30d / 90d 数据视图

### 5.2 SQL Playground 多 Tab 对比

1. `Ctrl+T` 新建 Tab，`Ctrl+W` 关闭
2. 双击 Tab 名可重命名
3. 在 2+ 个 Tab 执行查询后，点击「对比」按钮
4. 选择左右 Tab，查看差异高亮（匹配/差异/左独有/右独有）

### 5.3 血缘图谱交互

- 点击节点切换焦点，上下游路径高亮
- 拖拽节点调整布局
- 右上角 minimap 全局导航
- 缩放控制 50%-200%

### 5.4 设置页配置管理

- 7 个 Tab：通用 / Lint 规则 / 调度 / 通知 / 数据源 / 集成 / 高级
- 修改后显示「未保存 · N 处变更」badge
- YAML 导出：生成配置预览，可复制/下载
- YAML 导入：上传文件或粘贴文本

### 5.5 健康度异常归因

- 红表自动生成 3 列卡片：根因 / 下游影响 / 修复建议
- 修复步骤编号 + 箭头连接
- 显示最后出错时间 / 重试次数 / 预计修复时长

## 八、常见问题 FAQ

### Q1: 双击 index.html 后页面空白？

**A**: 可能是浏览器安全策略限制了 `file://` 协议的 JS 执行。解决方案：
- **Chrome**: 在地址栏输入 `chrome://flags/`，搜索 "Allow file access from files" 并启用
- **更推荐**: 使用本地 HTTP 服务：`bunx serve out -p 8080`，然后访问 `http://localhost:8080`
- **最简单**: 使用启动脚本 `start-dataops.bat` / `start-dataops.sh`

### Q2: 如何重新构建静态文件？

```bash
cd /path/to/my-project
bun run build:static
```

构建会覆盖 `out/` 目录，约 10 秒完成。

### Q3: 如何修改界面后生成新的离线版？

1. 修改 `src/` 下的源代码
2. 运行 `bun run build:static`
3. 双击 `out/index.html` 查看效果

### Q4: 离线版能看到实时日志吗？

离线版使用客户端模拟数据，能看到模拟的日志流效果。如需真实 WebSocket 日志推送，需使用开发模式（`bun run dev`）。

### Q5: 如何将离线版发给其他人？

将 `out/` 目录整个打包为 ZIP：
```bash
cd /path/to/my-project
zip -r dataops-offline.zip out/
```
对方解压后双击 `index.html` 即可使用。

### Q6: 支持哪些浏览器？

- ✅ Chrome 90+（推荐）
- ✅ Edge 90+
- ✅ Firefox 90+
- ✅ Safari 15+
- ❌ IE 不支持
