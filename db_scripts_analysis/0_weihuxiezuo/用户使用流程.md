## 一、快速开始

### 🚀 最快方式：双击打开

```
0_weihuxiezuo/dataops-ui/index.html  ← 双击这个文件，浏览器直接打开
```

**无需端口、无需启动服务器、无需安装任何软件**——这就是一个独立的 UI 界面。

### 三种打开方式

| 方式 | 操作 | 适用场景 |
|------|------|----------|
| **双击 HTML** | 双击 `dataops-ui/index.html` | ✅ 最简单，直接打开 |
| **启动脚本** | 运行 `start-dataops.bat` 或 `bash start-dataops.sh` | Windows 一键启动 |
| **HTTP 服务** | `python3 -m http.server 8080` → 访问 `http://localhost:8080` | Chrome 双击空白时用 |

> ⚠️ **Chrome 用户注意**：如果双击后页面空白，请改用 HTTP 服务方式（方式3），或用 Edge/Firefox 打开。

## 二、独立 UI 界面说明

`dataops-ui/` 目录是一个**完全独立的 UI 界面**，包含：

```
dataops-ui/
├── index.html          ← 主页面（双击打开）
├── _next/
│   └── static/
│       ├── chunks/     ← JavaScript 逻辑
│       └── media/      ← 字体文件
├── favicon.svg         ← 网站图标
├── logo.svg            ← Logo
└── robots.txt
```

**特性**：
- 📦 体积仅 ~2MB，可拷到任何电脑
- 🔌 完全离线，不依赖网络/端口/服务器
- 🎨 10 个功能视图全部可用
- 📊 所有数据为内置 Mock（展示用途）

## 三、首次构建（仅首次需要）

如果 `dataops-ui/` 目录不存在或需要更新：

```bash
# 1. 进入项目根目录
cd /path/to/my-project

# 2. 构建独立 UI（约 10 秒）
bun run build:static

# 3. 复制到交付目录
cp -r out db_scripts_analysis/0_weihuxiezuo/dataops-ui
```

构建完成后，`dataops-ui/` 就是一个独立的 UI 包，可以：
- 拷到任何电脑双击打开
- 打包成 ZIP 发给别人
- 放在 U 盘里随身携带

## 六、核心功能详解

### 6.1 Dashboard 钻取

- **数据表总数** → 点击跳转健康度视图
- **执行成功率** → 点击弹出 7 日成功率明细 Popover
- **入库行数** → 点击弹出 Top 5 大表 Popover
- **待处理告警** → 点击跳转规范校验视图
- **时间范围** → 切换 7d / 30d / 90d 数据视图

### 6.2 SQL Playground 多 Tab 对比

1. `Ctrl+T` 新建 Tab，`Ctrl+W` 关闭
2. 双击 Tab 名可重命名
3. 在 2+ 个 Tab 执行查询后，点击「对比」按钮
4. 选择左右 Tab，查看差异高亮（匹配/差异/左独有/右独有）

### 6.3 血缘图谱交互

- 点击节点切换焦点，上下游路径高亮
- 拖拽节点调整布局
- 右上角 minimap 全局导航
- 缩放控制 50%-200%

### 6.4 设置页配置管理

- 7 个 Tab：通用 / Lint 规则 / 调度 / 通知 / 数据源 / 集成 / 高级
- 修改后显示「未保存 · N 处变更」badge
- YAML 导出：生成配置预览，可复制/下载
- YAML 导入：上传文件或粘贴文本

### 6.5 健康度异常归因

- 红表自动生成 3 列卡片：根因 / 下游影响 / 修复建议
- 修复步骤编号 + 箭头连接
- 显示最后出错时间 / 重试次数 / 预计修复时长

## 八、常见问题 FAQ

### Q1: 双击 index.html 后页面空白？

Chrome 默认阻止 `file://` 协议加载 JS 模块。解决方案：

**方案 A（推荐）**：用本地 HTTP 服务
```bash
cd dataops-ui/
python3 -m http.server 8080
# 浏览器访问 http://localhost:8080
```

**方案 B**：用 Edge 或 Firefox 打开（这两个浏览器不限制 file:// 加载 JS）

**方案 C**：Chrome 启动时加参数
```bash
chrome --allow-file-access-from-files index.html
```

### Q2: 如何分发给其他人？

```bash
cd db_scripts_analysis/0_weihuxiezuo
zip -r dataops-ui.zip dataops-ui/
```

对方解压后双击 `index.html` 即可使用。

### Q3: 如何更新 UI？

修改源代码后重新构建：
```bash
cd /path/to/my-project
bun run build:static
cp -r out db_scripts_analysis/0_weihuxiezuo/dataops-ui
```

### Q4: 数据从哪来？

独立 UI 使用内置 Mock 数据，适合展示和汇报。如需连接真实数据库，需使用开发模式（`bun run dev`）并接入真实 API。

### Q5: 支持哪些浏览器？

| 浏览器 | 双击打开 | HTTP 服务 |

| Chrome 90+ | ⚠️ 可能空白 | ✅ 完美 |
